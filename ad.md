# 开发者，你还在用记事本管理那一堆密码吗？

---

## 场景够熟悉吗？

本地环境、测试环境、生产环境——同一个项目，三套域名，三个管理后台，三组账号密码。

再乘以你手头的项目数量。

还没算上数据库、服务器、第三方 API Key、云厂商控制台……

每次部署完切环境，打开记事本翻半天，复制出来的密码还不知道是哪个环境的。

---

## 你可能试过这些方案，但都不够好

**记事本 / 文本文件** — 没有层级，没有分类，搜索全靠 Ctrl+F，看到密文根本不知道对应哪个服务。

**网易云笔记 / 语雀** — 笔记内容明文存在对方服务器，你有多少把握他们不会泄露？况且一旦账号被盗，所有密码一锅端。

**Notion** — 国内访问时常抽风，关键时刻打不开，急死人。而且同样是云端明文存储。

**浏览器自带密码管理** — 只管密码，管不了域名备注、环境分组、相关文档，信息碎片化。

---

## 为多环境开发而生的私人信息管理器

**VaultKeeper** 是一款运行在你本机的私有信息管理工具，专为开发者设计，帮你把散落各处的项目信息——域名、账号、密码、配置备注——统一收纳、分层管理、随时检索。

### 多维度组织，告别信息乱堆

- **工作区隔离**：公司项目与个人项目各独立一个工作区，互不干扰
- **分组管理**：账号、书签按环境分组（本地 / 测试 / 生产），一目了然
- **标签联动**：跨类型打标签，搜 `#project-x` 立刻找到这个项目所有相关信息
- **书签 + 账号联动**：域名和密码放在一起，再也不用两个窗口来回找

### 本地运行，数据永远在你手里

VaultKeeper **不联网、不上云、不注册账号**。所有数据以加密数据库的形式存储在你自己的磁盘上，关掉软件什么也不留在网络上。

没有服务器，就没有拖库风险。没有账号，就没有撞库威胁。

### 军级加密，敏感内容再加一把锁

- **主密码加密**：整个数据库使用主密码加密，锁屏后无密码无法访问
- **笔记二次加密**：对特别敏感的笔记可单独设置二次密码，即使主密码泄露也无法读取
- **自动锁屏**：离开一段时间自动上锁，防止他人趁机翻看
